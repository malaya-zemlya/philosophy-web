The *bounded-table rejoinder* defends the possibility of a [lookup-table
mind](../concepts/lookup-table-mind.md) — a system that produces intelligent-seeming behaviour by sheer stored lookup, with a
pre-written response waiting for every input it might receive — against the charge that no finite
table could ever keep pace with a real mind. The objection it answers, the
[combinatorial-impossibility argument](lookup-table-combinatorial-impossibility.md), runs as
follows: the space of possible inputs is open-ended ([a mind
handles unbounded inputs](../claims/mind-handles-unbounded-inputs.md)) while any stored table is finite, so eventually an input must arrive on
which the mind responds appropriately and the table cannot. This rejoinder, due to Block
([block](../sources/block.md)), denies that the fatal divergence ever actually occurs.

### The inference

The move is a *rebuttal*: it concedes that the table is finite and attacks the asymmetry premise of
the combinatorial argument rather than its arithmetic.

1. [The table can be bounded to a lifetime
   of inputs](../claims/lifetime-bounded-table-covers-encountered-inputs.md). Stipulate it to store a response for every input string up to a length exceeding any
   the subject could receive in a finite life. On no input the subject *actually encounters* does it
   lack a stored answer.
2. [A finite mind also truncates beyond its capacity](../claims/finite-mind-truncates-beyond-capacity.md).
   On inputs past that bound — which the subject never meets — a finite mind no longer reliably
   out-responds the table; it too is capacity-limited.

It follows that the divergence the objection needs — an input on which the mind succeeds and the
table fails — never arises within the domain that matters. So combinatorial inadequacy
([lookup-table-combinatorially-inadequate](../claims/lookup-table-combinatorially-inadequate.md)) is not established: over the lived input space
the bounded table matches the behaviour, which is what [the
separability of behaviour from organisation](../claims/lookup-table-possible.md) requires. This restores the blockhead as a genuine
case of behaviour without organisation across the relevant domain.

### Where it is attacked

Both premises bear weight, and both draw fire.

Premise 1 is challenged on two fronts. On *physical* grounds, the
[physical-impossibility argument](lookup-table-physical-impossibility.md) notes that a
lifetime-bounded table is even larger than a real language model and simply cannot be built. On
*kind* grounds, the [pure-storage-instability reply](storage-not-stable-kind.md) holds that a
realisable table of that size is not "mere storage" at all but an organised computer. Premise 2 is
challenged by denying the levelling: a mind's degradation on over-long inputs is structural and
recursive, not the blind truncation a table performs, so the *manner* of competence still diverges
even where both eventually fail.

Distinct from [mind-handles-unbounded-inputs](../claims/mind-handles-unbounded-inputs.md) (the premise it denies): this is the argument
that denies that asymmetry, not the claim itself.

### In plain terms

This is Block defending the cheat-sheet machine ([lookup-table mind](../concepts/lookup-table-mind.md))
against the "a finite table can't keep up with an open-ended mind" reply
([the combinatorial-impossibility objection](lookup-table-combinatorial-impossibility.md)).
His move: you don't need a stored answer for *every conceivable* input — only for the ones a person
actually runs into across a whole lifetime. Within that range, the table always has a reply ready.
And the freakishly long or strange inputs beyond that bound? A real human mind breaks down on those
too — it isn't limitless either. So the moment where the mind shines and the table chokes never
actually comes up in real life. Across the inputs that matter, the table keeps pace with the person,
which puts the "behavior without a mind" case back in play. Where it's challenged: even granting the
coverage, could such a table physically exist (it's still astronomically huge?), and does a mind
really fail the *same way* a table does — or does it degrade more gracefully because it's genuinely
working things out rather than just looking them up?
