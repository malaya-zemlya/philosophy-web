The *relevance-filter regress* is an objection to the causal-structure reply. It grants that a system
has an objective causal structure, but argues that picking out the *mind-relevant* part of that
structure requires a further principle the reply has not supplied — so the demand for an
observer-independent invariant is not discharged, only relocated.

### The argument

The form is a *dilemma* that opens into a regress.

1. Raw causal structure includes every disposition — mass, thermal behaviour, the squishy noise a
   brain makes when struck — not only the cognitive ones
   ([physics-delivers-all-dispositions-indiscriminately](../claims/physics-delivers-all-dispositions-indiscriminately.md)).
2. If [functionalism](../concepts/functionalism.md) counts all of them, then the brain's structure is
   shared by no silicon system (which makes no squishy noise), multiple realizability fails, and the
   view collapses into physical chauvinism. So it must restrict to the *organisational*, mind-relevant
   subset.
3. But that restriction is a *further* principle, not delivered by causal structure itself. Selecting
   the relevant relations either presupposes what is mental (circularity) or requires an independent,
   observer-independent criterion — which is exactly the invariant at issue.

∴ The [causal-structure reply](causal-structure-supplies-invariant.md) does not discharge the
demand; it relocates it, so this argument supports [the
demand for a decomposition invariant](../claims/functionalism-owes-decomposition-invariant.md). Even the natural "difference-making" test — does removing this
change the computation? — presupposes the boundary of "the computation," which is the very carving in
question.

### The weakest premise

The vulnerable step is (3). A functionalist may offer a non-circular criterion of relevance — for
instance, control of the system's behavioural outputs, or information actually *used* downstream —
that excludes squishy noise without presupposing mentality. Whether any such criterion escapes both
chauvinism and circularity is the live crux, and it is the [role-grain
dilemma](role-grain-dilemma.md) under a new description.

This is distinct from [the
non-triviality-without-uniqueness reply](counterfactuals-yield-non-triviality-not-uniqueness.md): that reply grants the relevant relations and still finds
multiple carvings; this one questions the *selection* of the relevant relations in the first place.

### In plain terms

Another problem for the "cause-and-effect fixes the one true organization" reply
([causal-structure-supplies-invariant](causal-structure-supplies-invariant.md)). Physics hands you *all* of a system's
cause-and-effect, indiscriminately — not just the thinking-related bits, but its weight, its warmth,
the squelchy sound a brain makes if you poke it. Now you're stuck between two bad options. Count
*everything*, and a brain's full causal profile includes things no silicon chip shares (chips don't
squelch), so a machine can never match it — back to "only meat can think." To dodge that, you have to
keep only the *mind-relevant* causes and ignore the rest. But which causes are the mind-relevant
ones? Cause-and-effect by itself doesn't say — you need an extra rule to do the sorting, and that rule
either quietly assumes what already counts as "mental" (a circle) or *is* exactly the neutral,
observer-independent yardstick we were missing to begin with. So the reply doesn't settle the demand;
it just shifts it somewhere else. (A tempting test — "does removing this change the computation?" —
already assumes you've fixed where "the computation" starts and stops, which is the very thing in
dispute.)
