The *combinatorial-impossibility reply* argues that a finite lookup table cannot, even in
principle, match the behaviour of an open-ended mind, and so cannot serve as a counterexample to
the claim that behaviour suffices for mind. It is a reply to [the
lookup-table objection](lookup-table-objection.md), which uses a giant stored answer-table to argue that a system could
produce a mind's behaviour with "nobody home".

### The argument

The inference is at its core deductive — a diagonal or cardinality argument, that a fixed finite
domain cannot cover an unbounded input space — turning on one defeasible empirical premise (2).

1. For any finite lookup table there exist finite inputs outside its stored domain; to such an
   input it can at most respond by ignoring the part that exceeds its keys
   ([finite-table-misses-inputs](../claims/finite-table-misses-inputs.md)).
2. A genuine mind responds appropriately to such inputs rather than discarding their excess
   ([mind-handles-unbounded-inputs](../claims/mind-handles-unbounded-inputs.md)).

∴ For every finite table there is an input on which it diverges from the mind; so no finite table
reproduces a mind's full behavioural profile ([lookup-table-combinatorially-inadequate](../claims/lookup-table-combinatorially-inadequate.md)).

### What it attacks

The reply both responds to and attacks [the lookup-table
objection](lookup-table-objection.md) by denying its first premise, [that a finite table can
match the whole behavioural profile](../claims/lookup-table-possible.md). If a finite table cannot even produce the behaviour, the
blockhead is not a case of "behaviour without mind"; it is a system that fails to match the
behaviour in the first place. Dialectically this favours the substrate-independence side, blunting
the lookup-table objection to [behavioural parity](behavioral-parity.md) at its first
premise. It leaves untouched the Chinese-Nation objection
([china-nation-absent-qualia](china-nation-absent-qualia.md)), which supplies the organisation and so matches more
than bare input–output.

### Weakest premise

The vulnerable step is premise (2), [mind-handles-unbounded-inputs](../claims/mind-handles-unbounded-inputs.md). Block's "Blockhead"
([block](../sources/block.md)) is built to escape exactly this: stipulate a table covering all inputs up to a
length exceeding any the subject meets in a lifetime, and note that a finite mind also truncates
inputs past its own capacity — so the divergence asserted in premise (2) may never arise within
the bounded domain that matters. The conclusion is therefore filed `contested`. That escape is
[the bounded-table rejoinder](bounded-table-rejoinder.md), itself answered on physical
grounds by [the cosmic-size reply](lookup-table-physical-impossibility.md) (the bounded
table cannot be built) and on kind grounds by [storage-not-stable-kind](storage-not-stable-kind.md) (a realisable
table is not "mere storage").

### Distinct from neighbouring moves

It differs from [the lookup-table objection](lookup-table-objection.md) itself: that move
*uses* the lookup table to attack behavioural sufficiency (behaviour ⇒ mind), whereas this attacks
that move, denying the lookup table reproduces the behaviour at all — opposite dialectical
direction, different conclusion. It also differs from
[organisational-grain-dissolves-dilemma](organisational-grain-dissolves-dilemma.md), which excludes the blockhead by its lack of
internal *organisation*; this excludes it by its inability to match the *input–output profile*
over an unbounded input space, a different ground of exclusion.

### In plain terms

This pushes back on the cheat-sheet machine objection ([the
lookup-table objection](lookup-table-objection.md)). That whole worry assumed you *could* store a ready-made answer for
every possible input. But a real mind faces an open-ended world — there is no end to the fresh
sentences, situations, and questions it can sensibly deal with. Any finite stored table, however
gigantic, eventually meets an input nobody anticipated, and then all it can do is choke or ignore
the unfamiliar part. A genuine mind does not choke; it responds sensibly to the new thing. So no
finite lookup table can really keep pace with a mind across the board — which means the "behaviour
with nobody home" machine cannot even generate all the behaviour to begin with. The soft spot:
maybe you only ever need to cover the inputs a being actually meets in one finite lifetime, and a
real mind, too, gives out on inputs past *its* limits — which is the escape route the next
arguments go after.
