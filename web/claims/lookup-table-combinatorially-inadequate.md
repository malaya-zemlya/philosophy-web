gloss: Because any finite table misses some inputs ([finite-table-misses-inputs](finite-table-misses-inputs.md)) while a mind
responds appropriately to them ([mind-handles-unbounded-inputs](mind-handles-unbounded-inputs.md)), there is, for every finite
table, an input on which the two diverge. Hence no finite table matches a mind's *full* behavioural
profile: the table is combinatorially inadequate, not merely large.

scope: A claim about *finite* lookup tables only. It does NOT deny that some other non-mind could
match a mind, nor that a table could match a mind over a bounded fragment of inputs — only that
*pure finite lookup* cannot match the whole open-ended profile.

status — why contested: the conclusion is disputed at premise [mind-handles-unbounded-inputs](mind-handles-unbounded-inputs.md).
On Block's lifetime-bounded construction ([block](../sources/block.md)) the table need only cover inputs up to a
length the subject will actually receive, and a finite mind would itself truncate longer inputs — so
the asserted divergence may never arise in the domain that matters. The attacking move is
[bounded-table-rejoinder](../arguments/bounded-table-rejoinder.md) (Block); this claim's `contested` status records that dispute. The
bounded table is in turn answered by [lookup-table-physical-impossibility](../arguments/lookup-table-physical-impossibility.md) (it cannot be built)
and [storage-not-stable-kind](../arguments/storage-not-stable-kind.md) (a realisable table is not "mere storage").

who would deny it: a defender of Blockhead / pure input-output functionalism ([block](../sources/block.md)). Note
that this claim *helps* organisational-role functionalism: if true, it excludes the
lookup table from even matching behaviour, blunting [lookup-table-objection](../arguments/lookup-table-objection.md) at its first
premise.

Distinct from [lookup-table-lacks-mind](lookup-table-lacks-mind.md) (the table has behaviour but no mind): this denies the
table even has the *behaviour* — a stronger, prior claim. It is effectively the negation of
[lookup-table-possible](lookup-table-possible.md) (which asserts the finite table CAN reproduce the profile).
