A genuine mind can respond appropriately to inputs it has never met before — longer, more complex,
or recombined in new ways — by working through their structure rather than matching them to a
pre-stored whole. Its responsive capacity is *productive*, or open-ended: confronted with the very
inputs a finite table must miss ([finite-table-misses-inputs](finite-table-misses-inputs.md)), a mind does not simply
discard the excess.

### Scope

This is an *asymmetry* claim: a mind out-responds a finite table on inputs outside the table's
domain. It does not claim that minds are infinite or unbounded in capacity — only that for any
*fixed* finite table the mind's competence outruns it on some inputs.

### Who would deny it

A defender of Ned Block's "Blockhead" ([block](../sources/block.md)). Block stipulates a table covering every
input up to a length exceeding any the subject would receive in a normal lifetime, then argues
that (i) within that bounded domain the table matches the mind, and (ii) a real mind is *also*
finite — past its own memory and attention limits it too "ignores part of the input" — so the
asymmetry vanishes over the only domain that matters. This is the weakest link of
[the combinatorial-impossibility reply](../arguments/lookup-table-combinatorial-impossibility.md), and
why that argument's conclusion is held `contested`.

It should be distinguished from [behaviour insufficient for
mind](behaviour-insufficient-for-mind.md) (about whether matching behaviour suffices for mind): this is the prior, factual claim that
a finite table cannot even *match* an open-ended mind's behaviour. It differs too from
[role-grain-is-organisational](role-grain-is-organisational.md), which individuates minds by internal organisation; this
individuates by input–output competence over an unbounded input space, not by internals.

### In plain terms

A real mind keeps up with an endless variety of new inputs — sentences and situations it has never
met before — by actually working through their structure, not by recognising them off a stored
list. Hand it something longer or stranger than anything it has seen, and it can still answer
sensibly, where a fixed table just draws a blank. (It does not claim minds are limitless — only
that for any finite stored table, a mind can handle some inputs the table cannot.)
