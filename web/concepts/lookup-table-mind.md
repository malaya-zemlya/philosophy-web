A *lookup-table mind* is a system that produces all of a mind's behaviour by consulting a fixed,
stored map from input to canned response. Given any input it returns the pre-recorded right
output, but nothing happens in between: no inference, no mediating causal states, no internal
organisation beyond the table itself. The canonical version is Ned Block's "Blockhead," also
called the *Aunt Bubbles* machine ([block](../sources/block.md)): a device that holds a perfectly natural
conversation by storing, in advance, a fitting reply to every string its partner might utter.

### What distinguishes it

The defining feature, and what separates the lookup table from Block's other famous case, the
[Chinese Nation](chinese-nation.md), is that it *lacks internal functional organisation
altogether*. It is a flat query→answer map. It therefore reproduces only the *input–output
(behavioural) profile* of a mind, never the causal or transition structure — the web of internal
states that mediates between stimulus and response. The Chinese Nation, by contrast, does realise
that organisation; the lookup table deliberately strips it away.

### The dispute it feeds

The lookup table is a tool for one question: is producing a mind's behaviour enough to *have* a
mind? It drives the [lookup-table objection](../arguments/lookup-table-objection.md), which moves from
two premises — that such a table is possible ([lookup-table-possible](../claims/lookup-table-possible.md)) and that it
nonetheless has no mental states ([lookup-table-lacks-mind](../claims/lookup-table-lacks-mind.md)) — to the conclusion that
behaviour is not sufficient for mind ([behaviour-insufficient-for-mind](../claims/behaviour-insufficient-for-mind.md)). In its
contemporary use the objection is aimed at the abductive case from behavioural parity
([behavioral-parity](../arguments/behavioral-parity.md)) to the claim that large language models have mental states
([llm-has-mental-states](../claims/llm-has-mental-states.md)).

### Characteristic replies

Two counter-moves are specific to the lookup table, and differ from those aimed at the Chinese
Nation. The first is a *combinatorial-impossibility* reply: for open-ended, context-sensitive
behaviour the table would have to be not merely vast but infinite, so no finite lookup system
really matches a mind's full profile. The second is that the case is *cheaply excluded by
organisational-role functionalism* ([role-grain-is-organisational](../claims/role-grain-is-organisational.md)): because the table has
no internal organisation, an organisation-based theory denies it a mind for free. The objection
therefore embarrasses only behaviourism and pure input–output functionalism, not an
organisation-based view. The Chinese Nation, which *has* the organisation, is not excluded so
cheaply.

### In plain terms

Picture a machine that acts exactly like a person but pulls it off by cheating: for every possible
thing you could say to it, someone has pre-written the perfect reply in a gigantic table, and the
machine just looks up your input and reads the stored answer back. No thinking, no reasoning, no
inner steps — just an enormous answer key. That's the "lookup-table mind" (Block's "Blockhead").
It's a thought-experiment troublemaker: it would sail through any test that judges minds purely by
behavior, yet intuitively there's nobody in there. So it sharpens the question — is producing the
right behavior *enough* to count as having a mind, or could you get all the behavior with nobody
home? (Two standard replies: to really keep up with a real, open-ended mind the table would have
to be impossibly, near-infinitely large; and unlike a brain it has no inner organization at all,
so it only embarrasses theories that judge minds by behavior alone.)
